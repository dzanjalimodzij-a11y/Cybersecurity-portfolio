# Access Control Policy

## Purpose

This policy sets rules for creating, changing, reviewing, and removing access to BrightBox systems.

## Scope

This policy applies to Microsoft 365, Shopify, cloud storage, shared mailboxes, finance systems, staff laptops, and supplier access.

## Policy requirements

- Access must be approved before it is granted.
- Users should receive the minimum access needed for their role.
- Joiner, mover, and leaver changes must be requested by the operations manager and completed by the IT support provider.
- Admin access must be limited to named users and used only when needed.
- Shared accounts should be avoided. Where shared mailboxes are needed, named users must have individual access.
- Leaver access must be removed on or before the last working day.
- Access reviews must be completed quarterly for Microsoft 365, Shopify, finance systems, and shared folders.
- Supplier access must have an owner, purpose, and review date.

## Roles and responsibilities

| Role | Responsibilities |
|------|------------------|
| Director | Approves high-risk or admin access. |
| Operations Manager | Requests access changes and completes reviews. |
| IT Support Provider | Implements access changes and keeps technical records. |
| Staff | Use only their own accounts and report access issues. |

## Evidence or records

- access request records
- leaver checklist
- access review record
- admin account list
- supplier access list

## Review frequency

This policy should be reviewed annually. Access reviews should be completed quarterly.
