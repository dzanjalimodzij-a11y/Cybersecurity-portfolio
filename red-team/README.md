# Red Team Projects

This section documents offensive security projects conducted exclusively within controlled, authorised environments. All work was performed on intentionally vulnerable practice platforms or isolated local virtual machine labs built for educational purposes.

> ⚠️ **Ethical Notice:** No real organisations, systems, or individuals were targeted. All techniques documented here are presented in an educational context to demonstrate understanding of attack methods and their defensive countermeasures.

---

## Projects

### 1. [OWASP Juice Shop Web Application Assessment](./owasp-juice-shop-assessment/)

A structured web application penetration test against OWASP Juice Shop — a deliberately vulnerable application designed for security training. Covers the OWASP Top 10, structured finding documentation, and professional report writing.

**Key skills:** Web app pentesting, Burp Suite, OWASP Top 10, vulnerability documentation, remediation advice.

---

### 2. [Active Directory Attack Path Lab](./active-directory-attack-path-lab/)

A local lab simulating an Active Directory environment with intentional misconfigurations. Documents the process of enumerating the environment, identifying attack paths, and mapping findings to the MITRE ATT&CK framework — with a defensive recommendations section.

**Key skills:** AD enumeration, privilege escalation paths, MITRE ATT&CK, BloodHound, defensive hardening.

---

## Approach

Each red team project follows a structured methodology:

1. Define scope and rules of engagement before any testing begins
2. Conduct reconnaissance and enumeration
3. Identify vulnerabilities and attack paths
4. Document findings with clear evidence, impact ratings, and remediation advice
5. Produce a professional report suitable for a technical or management audience

The goal is not just to find vulnerabilities — it is to demonstrate how they would be communicated and remediated in a professional engagement.
