## Introduction

This repository is a structured portfolio documenting practical cybersecurity work across three core disciplines: **Blue Team (Detection & Response)**, **Red Team (Offensive Security)**, and **Governance, Risk and Compliance (GRC)**. Each project reflects hands-on work completed in controlled lab environments or as structured independent practice exercises.

All offensive security work documented here was conducted exclusively in isolated local lab environments or explicitly authorised platforms. No real systems, networks, or organisations were targeted.

## Portfolio Goals

This portfolio was built with the following goals:

1. **Practise practical ability** across the three main pillars of cybersecurity — detection, offence, and governance — through realistic, scenario-driven work.
2. **Show structured, professional thinking** in how projects are scoped, executed, documented, and reviewed.
3. **Bridge academic learning and workplace readiness** by applying course knowledge to realistic, scenario-driven projects.
4. **Develop reusable, realistic artefacts** such as incident reports, risk registers, penetration testing reports, and policy documents.
5. **Maintain ethical and legal standards** in all work — particularly in offensive security — by operating only within controlled, authorised environments.

## Skills Demonstrated

### Technical Skills

- Security information and event management (SIEM) — log ingestion, alert triage, detection rule creation
- Endpoint detection using Windows Sysmon and Event Logs

- Indicator of Compromise (IOC) extraction and documentation

- Control mapping against recognised frameworks
- Executive-level summary writing

### Professional Skills

- Structured technical report writing
- Incident response documentation and playbook development
- Scope definition and rules of engagement for assessments
- Clear, audience-appropriate communication (technical and non-technical)

## Ethical and Legal Notice

All offensive security work documented in this portfolio — including penetration testing techniques, vulnerability exploitation, and Active Directory attack simulations — was performed **exclusively within isolated local virtual machine environments** or on **deliberately vulnerable, publicly available practice platforms** (such as OWASP Juice Shop) that are designed for educational use.

- No real organisations, systems, networks, or individuals were targeted.
- No unauthorised access was attempted or achieved.
- All activities comply with applicable laws and ethical standards.
- Where tools with dual-use potential are referenced, they are discussed in a strictly educational and defensive context.

If you have any questions about the scope or ethics of any project, please feel free to reach out.
