# Backup Policy

## Purpose

This policy defines how BrightBox backs up important business information and tests recovery.

## Scope

This policy applies to Microsoft 365 data, cloud file storage, Shopify-related records, finance records, customer support records, and important business documents.

## Policy requirements

- Important business data must be backed up using approved services.
- Microsoft 365 and cloud storage backups should be reviewed with the IT support provider.
- Backup frequency should match business need, with key cloud data backed up at least daily where the service supports it.
- Backup access must be restricted to authorised users.
- Restore tests must be completed at least twice a year.
- Backup results and restore tests must be recorded.
- Backup arrangements should consider ransomware, accidental deletion, and supplier outage scenarios.
- Local laptop data should be avoided where possible. Staff should store work in approved cloud storage.

## Roles and responsibilities

| Role | Responsibilities |
|------|------------------|
| Director | Approves backup approach and accepts residual risk. |
| IT Support Provider | Configures backups, monitors issues, and runs restore tests. |
| Operations Manager | Keeps restore test records and tracks follow-up actions. |
| Staff | Store work in approved locations so it can be protected. |

## Evidence or records

- backup configuration summary
- backup logs or status reports
- restore test record
- backup issue log

## Review frequency

This policy should be reviewed annually. Restore tests should be completed twice yearly.
