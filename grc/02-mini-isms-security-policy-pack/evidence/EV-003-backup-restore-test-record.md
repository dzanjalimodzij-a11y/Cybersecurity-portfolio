# EV-003 - Backup Restore Test Record

## Evidence ID

EV-003

## Evidence title

Backup restore test record

## Related policy or register

[Backup Policy](../policies/backup-policy.md)

## Control area

Backup and recovery testing

## Example record details

| Field | Example entry |
|-------|---------------|
| System tested | Microsoft 365 cloud file storage |
| Backup type | Cloud backup of shared document library |
| Test date | 2026-06-14 |
| Test item | Restore one deleted supplier contract file |
| Restore result | File restored successfully to test folder |
| Time to restore | 18 minutes |
| Issues found | Restore permissions needed clarification |
| Corrective action | Document restore approval process and backup administrator contact |
| Owner | IT Support Provider |

## Why this evidence is important

Backup records alone do not prove that recovery will work. Restore testing gives confidence that the company can recover important files after deletion, ransomware, or service issues.

## Review or collection frequency

Collect at least twice a year and after major backup configuration changes.

## Notes on limitations

This test covers one restore scenario only. Larger recovery tests would be needed for wider outage planning.
