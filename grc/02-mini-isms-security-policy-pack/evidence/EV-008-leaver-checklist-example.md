# EV-008 - Leaver Checklist Example

## Evidence ID

EV-008

## Evidence title

Leaver checklist example

## Related policy or register

[Access Control Policy](../policies/access-control-policy.md)

## Control area

Leaver access removal and asset return

## Example record details

| Check | Owner | Status | Notes |
|-------|-------|--------|-------|
| Microsoft 365 account disabled | IT Support Provider | Complete | Disabled on final working day |
| Shopify access removed | Operations Manager | Complete | Access removed from admin users list |
| Cloud storage access removed | IT Support Provider | Complete | User removed from shared folders |
| Device returned | Operations Manager | Complete | Laptop returned and checked in |
| Shared password review | Operations Manager | Complete | No shared passwords identified |
| Email forwarding/removal decision | Operations Manager | Complete | Mailbox converted to inactive mailbox for retention |
| MFA method removed | IT Support Provider | Complete | Authenticator method removed |

## Why this evidence is important

Leaver evidence helps show that access is removed promptly and company assets are returned. This reduces the risk of unused accounts, data exposure, and missing devices.

## Review or collection frequency

Complete for every leaver and review as part of the quarterly access review.

## Notes on limitations

The checklist depends on the operations manager being told about leavers in time. It should be linked to the HR or management leaver process.
